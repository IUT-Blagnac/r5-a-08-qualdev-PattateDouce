  1. Mettez à jour votre pom.xml avec les trois dépendances suivantes :

<dependencies>
    <dependency> (1)
        <groupId>io.cucumber</groupId>
        <artifactId>cucumber-java</artifactId>
        <version>3.0.2</version>
        <scope>test</scope>
    </dependency>
    <dependency> (2)
        <groupId>io.cucumber</groupId>
        <artifactId>cucumber-junit</artifactId>
        <version>3.0.2</version>
        <scope>test</scope>
    </dependency>
    <dependency> (3)
        <groupId>junit</groupId>
        <artifactId>junit</artifactId>
        <version>4.12</version>
        <scope>test</scope>
    </dependency>
</dependencies>
1 Pour pouvoir utiliser Cucumber.
2 Pour pouvoir exécuter les scénarios Cucumber dans JUnit à l’aide du JUnit Runner.
3 Pour Junit (si besoin).

Une fois, le pom.xml modifié, n’oubliez pas de relancer Maven  Update Project… dans eclipse (ou l’équivalent dans votre IDE). En ligne de commande, tapez : mvn clean install -U

  1. Ajoutez le fichier cocktail.feature dans un package dojo de src/main/resources :

  Scenario: Creating an empty order

  1. Créez le fichier pour lancer les tests en lien avec la feature, par exemple RunCucumberTest.java dans un package dojo de src/test/java.

import org.junit.runner.RunWith;

@RunWith(Cucumber.class)  (1)
public class RunCucumberTest { }

  1. En utilisant les 3 implémentations proposées ci-dessous pour les 3 steps, proposer un code qui permette de passer les tests.

Aidez-vous des fonctionalités de votre IDE (quick tips pour créer les classes et méthodes absentes par exemple).
On met en oeuvre ici le "Behavior-Driven". En effet, le développement (code de la feature) est dirigé par les test d’acceptances. 
public class CocktailSteps {
    private Order order;
    @Given("Romeo who wants to buy a drink")
    public void romeo_who_wants_to_buy_a_drink() {
         order = new Order();
         order.declareOwner("Romeo");
    }
    //...
}
public class CocktailSteps {
    //...
    @When("an order is declared for Juliette")
           public void an_order_is_declared_for_juliette() {
            order.declareTarget("Juliette");
    }
    //...
}
import static org.junit.Assert.*;
import java.util.List;

public class CocktailSteps {
    //...
    @Then("there is no cocktail in the order")
    public void there_is_no_cocktail_in_the_order() {
        List<String> cocktails =  order.getCocktails();
        assertEquals(0, cocktails.size());
    }
}

Il faut dans le code Java du step :

  • ajouter l’expression régulière (.*) (ou {string}) à la place de Romeo dans l’annotation Given ;

  • ajouter un paramètre de type String x dans la signature de la méthode ;

  • passer le paramètre x dans la méthode declareOwner à la place de la valeur "Romeo".
